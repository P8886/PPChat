# PPChat 数据库设计文档

## 数据库概览

PPChat 使用 Supabase 提供的 PostgreSQL 数据库，采用行级安全策略 (RLS) 控制数据访问权限，支持实时订阅。

## 数据表详细设计

### 1. 用户表 (users)

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | uuid | PK, FK → auth.users | 用户ID，关联 Supabase Auth |
| username | text | UNIQUE | 用户名，用于登录 |
| status | user_status | DEFAULT 'OFFLINE' | 用户在线状态 |

**关联关系：**
- `id` → `auth.users.id` (Supabase 认证用户)
- 被引用：`channels.created_by`、`messages.user_id`、`user_roles.user_id`

---

### 2. 频道表 (channels)

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | bigint | PK, AUTO | 频道ID |
| inserted_at | timestamptz | DEFAULT now() | 创建时间 |
| slug | text | NOT NULL, UNIQUE | 频道标识名 |
| created_by | uuid | FK → users.id | 创建者ID |
| is_private | boolean | DEFAULT false | 是否为私密频道（需要密码访问） |
| password | text | - | 私密频道密码（明文存储） |

**关联关系：**
- `created_by` → `users.id`
- 被引用：`messages.channel_id`

**私密频道功能：**
- 私密频道在 UI 中显示 🔒 图标
- 用户需要输入正确密码才能访问私密频道内容
- 已验证的私密频道 ID 存储在浏览器 localStorage 中
- 未验证密码时，私密频道的消息提示音和小红点不会触发

---

### 3. 消息表 (messages)

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | bigint | PK, AUTO | 消息ID |
| inserted_at | timestamptz | DEFAULT now() | 发送时间 |
| message | text | - | 消息内容 / 图片URL |
| user_id | uuid | FK → users.id, NOT NULL | 发送者ID |
| channel_id | bigint | FK → channels.id, NOT NULL | 所属频道ID |
| message_type | text | DEFAULT 'text' | 消息类型：text / image |
| file_hash_id | bigint | FK → file_hashes.id | 关联的文件哈希ID（用于图片去重） |

**关联关系：**
- `user_id` → `users.id`
- `channel_id` → `channels.id` (CASCADE DELETE)
- `file_hash_id` → `file_hashes.id`

---

### 4. 文件哈希表 (file_hashes)

用于图片文件去重和引用计数管理。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | bigint | PK, AUTO | 记录ID |
| hash_value | text | NOT NULL, UNIQUE | 文件 SHA-256 哈希值 |
| file_path | text | NOT NULL | 文件在 Storage 中的路径 |
| file_size | bigint | - | 文件大小（字节） |
| file_type | text | - | 文件 MIME 类型 |
| reference_count | integer | DEFAULT 1 | 引用计数（消息中引用此文件的次数） |
| created_at | timestamptz | DEFAULT now() | 创建时间 |
| updated_at | timestamptz | DEFAULT now() | 更新时间 |

**功能说明：**
- 上传图片时计算 SHA-256 哈希值
- 如果哈希值已存在，增加引用计数，复用已有文件
- 删除消息时减少引用计数
- 引用计数为 0 时，从 Storage 中删除实际文件

---

### 5. 用户角色表 (user_roles)

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | bigint | PK, AUTO | 记录ID |
| user_id | uuid | FK → users.id, NOT NULL | 用户ID |
| role | app_role | NOT NULL | 角色类型：admin / moderator |

**唯一约束：** (user_id, role)

**关联关系：**
- `user_id` → `users.id` (CASCADE DELETE)

---

### 6. 角色权限表 (role_permissions)

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | bigint | PK, AUTO | 记录ID |
| role | app_role | NOT NULL | 角色类型 |
| permission | app_permission | NOT NULL | 权限类型 |

**唯一约束：** (role, permission)

---

## 枚举类型

### app_permission (权限类型)
| 值 | 说明 |
|------|------|
| channels.delete | 删除频道权限 |
| messages.delete | 删除消息权限 |

### app_role (角色类型)
| 值 | 说明 |
|------|------|
| admin | 管理员 |
| moderator | 版主 |

### user_status (用户状态)
| 值 | 说明 |
|------|------|
| ONLINE | 在线 |
| OFFLINE | 离线 |

### Supabase Storage Buckets

| Bucket 名称 | 用途 | 访问权限 |

```

│   ├── {timestamp}_{random}.jpg
│   ├── {timestamp}_{random}.png
│   └── {timestamp}_{random}.gif

```

### Storage 与数据库关系

```
┌────────────────────┐         ┌────────────────────┐
│  chat-images       │         │     messages       │

```

## RLS 策略汇总

| 表名 | 操作 | 策略 |
|------|------|------|
| users | SELECT | 已登录用户可读 |
| users | INSERT | 仅自己可插入 |
| users | UPDATE | 仅自己可更新 |
| channels | SELECT | 已登录用户可读 |
| channels | INSERT | 创建者可插入 |
| channels | DELETE | 创建者或有权限者可删 |
| messages | SELECT | 已登录用户可读 |
| messages | INSERT | 发送者可插入 |
| messages | UPDATE | 发送者可更新 |
| messages | DELETE | 发送者或有权限者可删 |
| user_roles | SELECT | 仅自己可读 |
| file_hashes | SELECT | 已登录用户可读 |
| file_hashes | INSERT | 已登录用户可插入 |
| file_hashes | UPDATE | 已登录用户可更新 |
| file_hashes | DELETE | 已登录用户可删除 |

### Storage RLS 策略

| 操作 | 策略 |
|------|------|
| SELECT | 公开读取 |
| INSERT | 认证用户可上传 |
| UPDATE | 仅自己可更新 |
| DELETE | 仅自己可删除 |

## 前端功能特性

### 1. 消息提示音

- **触发条件**：收到其他用户的新消息时播放
- **排除情况**：
  - 自己发送的消息不播放
  - 未验证密码的私密频道消息不播放
  - 页面在前台时不播放（仅后台时播放）
- **音效**：三声"嘀嘀嘀"（880Hz 正弦波）

### 2. 页面可见性检测

- **检测方式**：结合 `visibilitychange` 事件和 `document.hasFocus()`
- **功能**：
  - 页面切换到后台时记录隐藏时间
  - 页面回到前台时清除当前频道未读状态
  - 超过 5 分钟隐藏后，检测连接健康状态

### 3. 连接健康检测

- **触发时机**：页面从后台返回且隐藏超过 5 分钟
- **检测方式**：向 `channels` 表发送心跳查询
- **失败处理**：显示黄色提示条，提示用户刷新页面

### 4. 乐观更新机制

解决长时间停留页面后 Realtime 连接断开，导致发送消息不显示的问题。

**实现原理：**
1. 用户点击发送后，**立即在本地显示消息**（临时消息，ID 前缀为 `temp-`）
2. 同时向服务器发送请求
3. 收到服务器确认后，用真实消息替换临时消息
4. 如果 Realtime 推送了相同内容的消息，自动去重

**流程图：**
```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 点击发送 │───▶│ 立即显示 │───▶│ 发送请求 │───▶│ 收到确认 │
└──────────┘    │ 临时消息 │    │ 到服务器 │    │ 替换消息 │
                └──────────┘    └──────────┘    └──────────┘
                       │                              │
                       │         Realtime 推送         │
                       └─────────── 去重处理 ───────────┘
```

**好处：**
- 用户无需等待，消息"秒发"
- 即使 Realtime 断开，消息也不会"消失"

### 5. 未读消息提示

- **标题栏**：未读消息数量显示为 `🔴 (n) PPChat`
- **侧边栏**：频道列表中未读频道显示红色圆点 ●
- **排除规则**：未验证密码的私密频道不计入未读数

### 6. 本地存储项

| 键名 | 用途 | 数据格式 |
|------|------|----------|
| `ppchat_accessible_private` | 已验证密码的私密频道ID列表 | JSON 数组 |
| `ppchat_last_channel` | 上次访问的频道ID | 字符串 |

### 7. 频道排序

- 按创建时间降序排列（最新的频道在最上面）
- 新创建的频道实时添加到列表顶部
