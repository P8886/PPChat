# PPChat 数据库设计文档

## 数据库概览

PPChat 使用 Supabase 提供的 PostgreSQL 数据库，采用行级安全策略 (RLS) 控制数据访问权限，支持实时订阅。

## 数据表详细设计

### 1. 用户表 (users)

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | uuid | PK, FK → auth.users | 用户ID，关联 Supabase Auth |
| username | text | UNIQUE | 用户名，用于登录 |
| status | user_status | DEFAULT 'OFFLINE' | 用户在线状态 |

**关联关系：**
- `id` → `auth.users.id` (Supabase 认证用户)
- 被引用：`channels.created_by`、`messages.user_id`、`user_roles.user_id`

---

### 2. 频道表 (channels)

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | bigint | PK, AUTO | 频道ID |
| inserted_at | timestamptz | DEFAULT now() | 创建时间 |
| slug | text | NOT NULL, UNIQUE | 频道标识名 |
| created_by | uuid | FK → users.id | 创建者ID |
| is_private | boolean | DEFAULT false | 是否为私密频道 |
| password | text | - | 私密频道密码 |

**关联关系：**
- `created_by` → `users.id`
- 被引用：`messages.channel_id`

---

### 3. 消息表 (messages)

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | bigint | PK, AUTO | 消息ID |
| inserted_at | timestamptz | DEFAULT now() | 发送时间 |
| message | text | - | 消息内容 / 图片URL |
| user_id | uuid | FK → users.id, NOT NULL | 发送者ID |
| channel_id | bigint | FK → channels.id, NOT NULL | 所属频道ID |
| message_type | text | DEFAULT 'text' | 消息类型：text / image |

**关联关系：**
- `user_id` → `users.id`
- `channel_id` → `channels.id` (CASCADE DELETE)

---

### 4. 用户角色表 (user_roles)

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | bigint | PK, AUTO | 记录ID |
| user_id | uuid | FK → users.id, NOT NULL | 用户ID |
| role | app_role | NOT NULL | 角色类型：admin / moderator |

**唯一约束：** (user_id, role)

**关联关系：**
- `user_id` → `users.id` (CASCADE DELETE)

---

### 5. 角色权限表 (role_permissions)

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | bigint | PK, AUTO | 记录ID |
| role | app_role | NOT NULL | 角色类型 |
| permission | app_permission | NOT NULL | 权限类型 |

**唯一约束：** (role, permission)

---

## 枚举类型

### app_permission (权限类型)
| 值 | 说明 |
|------|------|
| channels.delete | 删除频道权限 |
| messages.delete | 删除消息权限 |

### app_role (角色类型)
| 值 | 说明 |
|------|------|
| admin | 管理员 |
| moderator | 版主 |

### user_status (用户状态)
| 值 | 说明 |
|------|------|
| ONLINE | 在线 |
| OFFLINE | 离线 |

## 存储结构

### Supabase Storage Buckets

| Bucket 名称 | 用途 | 访问权限 |
|-------------|------|----------|
| chat-images | 聊天图片存储 | Public (公开读取) |

### 存储路径规则

```
chat-images/
├── {user_id}/           # 用户文件夹
│   ├── {timestamp}_{random}.jpg
│   ├── {timestamp}_{random}.png
│   └── {timestamp}_{random}.gif
└── ...
```

### Storage 与数据库关系

```
┌────────────────────┐         ┌────────────────────┐
│  chat-images       │         │     messages       │
│  (Storage Bucket)  │         │     (数据库表)      │
├────────────────────┤         ├────────────────────┤
│ 文件实际存储        │  ────▶  │ message 字段存储    │
│ user_id/xxx.jpg    │   URL   │ 图片公开访问地址    │
└────────────────────┘         └────────────────────┘
```

## RLS 策略汇总

| 表名 | 操作 | 策略 |
|------|------|------|
| users | SELECT | 已登录用户可读 |
| users | INSERT | 仅自己可插入 |
| users | UPDATE | 仅自己可更新 |
| channels | SELECT | 已登录用户可读 |
| channels | INSERT | 创建者可插入 |
| channels | DELETE | 创建者或有权限者可删 |
| messages | SELECT | 已登录用户可读 |
| messages | INSERT | 发送者可插入 |
| messages | UPDATE | 发送者可更新 |
| messages | DELETE | 发送者或有权限者可删 |
| user_roles | SELECT | 仅自己可读 |

### Storage RLS 策略

| 操作 | 策略 |
|------|------|
| SELECT | 公开读取 |
| INSERT | 认证用户可上传 |
| UPDATE | 仅自己可更新 |
| DELETE | 仅自己可删除 |

```
┌──────────┐    ┌──────────┐    ┌──────────┐
│ 注册请求 │───▶│ Auth用户 │───▶│ 触发器   │
└──────────┘    │ 创建     │    │ handle_  │
                └──────────┘    │ new_user │
                                └────┬─────┘
                                     │
                    ┌────────────────┼────────────────┐
                    ▼                ▼                ▼
              ┌──────────┐    ┌──────────┐    ┌──────────┐
              │ users 表 │    │user_roles│    │ 角色判断 │
              │ 插入记录 │    │ 分配角色 │    │(邮箱后缀)│
              └──────────┘    └──────────┘    └──────────┘
```
